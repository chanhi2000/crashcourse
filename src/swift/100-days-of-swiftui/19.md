###  Your challenge

You need to build an app that handles unit conversions: users will select an input unit and an output unit, then enter a value, and see the output of the conversion.

Which units you choose are down to you, but you could choose one of these:

- Temperature conversion: users choose Celsius, Fahrenheit, or Kelvin.
- Length conversion: users choose meters, kilometers, feet, yards, or miles.
- Time conversion: users choose seconds, minutes, hours, or days.
- Volume conversion: users choose milliliters, liters, cups, pints, or gallons.

If you were going for length conversion you might have:

- A segmented control for meters, kilometers, feet, yard, or miles, for the input unit.
- A second segmented control for meters, kilometers, feet, yard, or miles, for the output unit.
- A text field where users enter a number.
- A text view showing the result of the conversion.

So, if you chose meters for source unit and feet for output unit, then entered 10, you‚Äôd see 32.81 as the output.

__You know almost everything you need to build this.__ In fact, if you go for a simple solution ‚Äì and I hope you do ‚Äì the only actually new thing you‚Äôll find is that you need to use a different `format` option for your text field, because `.currency` and `.percent` don‚Äôt make sense here. You should choose `.number` instead, which is suitable for standard numerical input.

### Tips

You know everything you need to complete this project, but if you‚Äôre hitting problems then I have some tips that might help.

First, and most importantly: keep it simple! You‚Äôll have lots of time in your future career to think up clever coding architectures and more, but in this initial challenge I want you to go for the simplest solution you can find.

Second, all our unit conversions are simple mathematics, but you shouldn‚Äôt try to write conversions to go from every source unit to very other unit. A better idea is to convert the user‚Äôs input to a single base unit, then convert from there to the target unit.

So, rather than writing code to convert from liters to milliliters, and from liters to cups, and from liters to pints, and so on, a better idea is to convert the user‚Äôs input into milliliters (the lowest common denominator), then convert from there to whatever output unit they want.

Third, as we have three user values here ‚Äì their input number, their input unit, and their output unit ‚Äì you need to have three `@State` properties to store them all. You‚Äôll need a `TextField`, two pickers, and a text view to show your output, and that‚Äôs about it. You can break your form up into sections if you want, but it‚Äôs not required.

Fourth, the easiest way to store your conversion units is to have a simple string array, which you can loop over using something like `ForEach(units, id: \.self)`.

Finally, if you‚Äôd like to be a bit fancy with the output number you display, try calling `.formatted()` on it ‚Äì e.g. `someDouble.formatted()`. This will cause iOS to format the number for printing, adding thousands separator and removing a lot of the noise from unnecessary decimal places.
